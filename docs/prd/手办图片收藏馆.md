AI 手办图片收藏馆 PRD

1.2 项目定位

这是一个面向个人使用的本地桌面图片收藏管理客户端。

项目不负责 AI 图片生成，只负责管理用户电脑中已经存在的 AI 处理图片。主要使用场景是：用户已经在电脑里保存了大量 AI 手办化图片，但文件夹混乱、查找困难、缺少分类和对比展示能力，因此需要一个本地收藏墙工具进行整理、浏览和展示。

1.3 一句话描述

一个基于 Tauri 2 + React 的个人本地 AI 手办图片收藏客户端，用收藏墙整理电脑中散乱的图片，支持相册、标签、星标、单个收藏项导入，以及 Before / After 滑杆和橡皮擦对比展示。

⸻

2. 项目目标

2.1 核心目标

1. 将电脑中散乱的 AI 手办图片集中管理。
2. 用收藏墙形式快速浏览图片。
3. 支持相册 / 系列管理。
4. 支持简单标签和搜索。
5. 支持单图收藏和 Before / After 双图收藏。
6. 支持图片对比特效，包括滑杆对比和橡皮擦对比。
7. 所有数据本地保存，不依赖云端服务。

2.2 非目标

第一版不做以下功能：

1. AI 图片生成。
2. Prompt 保存和管理。
3. 用户登录。
4. 云同步。
5. 多设备同步。
6. 批量导入。
7. 多人协作。
8. 复杂权限系统。
9. 回收站。
10. 自动识别图片内容。
11. 在线分享。
12. 复杂图库编辑功能。

⸻

3. 目标用户

3.1 主要用户

项目仅面向个人使用。

3.2 用户特点

1. 图片已经在本地电脑中。
2. 图片数量较多，当前文件管理比较混乱。
3. 需要快速浏览和整理 AI 手办图片。
4. 需要收藏墙式展示。
5. 需要对比 AI 处理前后的效果。

例如：

1. 可爱

用户可以给喜欢的收藏项打星标，后续可以筛选只看收藏。

6.2 单个收藏项导入

6.2.1 功能描述

用户可以通过导入弹窗创建一个收藏项。

当前版本一次创建一个收藏项。

每个收藏项支持 1 到 2 张图片：

1. After 图片：必填。
2. Before 图片：可选。

6.2.2 表单字段

1. After 图片，必填。
2. Before 图片，可选。
3. 标题，必填。
4. 描述，可选。
5. 相册，可选。
6. 分类，可选。
7. 标签，可选。
8. 是否星标，可选。

6.2.3 导入流程

1. 用户点击“导入图片”按钮。
2. 打开导入弹窗。
3. 用户选择 After 图片。
4. 用户可选选择 Before 图片。
5. 用户填写标题、描述、相册和标签。
6. 用户点击保存。
7. 应用将图片复制到应用数据目录。
8. 应用写入 SQLite 数据库。
9. 收藏墙刷新展示新收藏项。

6.2.4 图片保存规则

导入后复制到：

images/{figureId}/after.{ext}
images/{figureId}/before.{ext}

其中：

1. after 为必填主图。
2. before 为可选原图。
3. 原图路径不作为长期依赖。
4. 删除收藏项时，删除应用目录中的图片副本。

6.4 Before / After 滑杆对比

6.4.1 功能描述

用户可以通过滑杆横向拖动，查看 Before 图和 After 图之间的差异。

6.4.2 展示逻辑

1. 底层展示 Before 图。
2. 上层展示 After 图。
3. 通过滑杆改变上层图显示宽度。
4. 默认滑杆位置为 50%。

6.4.3 交互

1. 用户左右拖动滑杆。
2. 拖动时实时改变图片显示比例。
3. 可以双击回到 50%，该功能可放到后续版本。

6.4.4 注意事项

Before 和 After 图片最好使用相同或接近的宽高比例，否则对比可能出现错位。

6.5 橡皮擦对比

6.5.1 功能描述

橡皮擦对比是一种更有趣的 Before / After 展示方式。

用户可以像擦橡皮一样擦掉上层图片，被擦掉的位置露出下层图片。

6.5.2 展示逻辑

1. 底层展示 Before 图。
2. 上层 Canvas 绘制 After 图。
3. 用户鼠标或触控拖动时，通过 Canvas 擦除上层区域。
4. 被擦除区域露出下层 Before 图。

6.5.3 交互

第一版支持：

1. 鼠标按下开始擦除。
2. 鼠标移动持续擦除。
3. 鼠标松开结束擦除。
4. 触控拖动擦除。
5. 重置按钮。
6. 画笔大小调整。

6.5.4 后续增强

1. 反向擦除。
2. 一键全擦。
3. 一键恢复。
4. 画笔形状切换。
5. 保存擦除结果为图片。

7. 数据库设计

7.1 设计原则

1. 支持当前最多两张图。
2. 支持未来扩展多张图。
3. 收藏项与图片分表。
4. 相册独立成表。
5. 标签独立成表。
6. 收藏项和标签使用多对多关系表。
7. 删除收藏项时级联删除图片和标签关联。

7.2 figures 表

收藏项主体表。

CREATE TABLE figures (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  description TEXT,
  album_id TEXT,
  category TEXT,
  rating INTEGER DEFAULT 0,
  is_favorite INTEGER DEFAULT 0,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (album_id) REFERENCES albums(id)
);

字段说明：

字段	类型	必填	说明
id	TEXT	是	收藏项 ID
title	TEXT	是	标题
description	TEXT	否	描述
album_id	TEXT	否	所属相册
category	TEXT	否	简单分类
rating	INTEGER	否	评分，默认 0
is_favorite	INTEGER	否	是否星标
created_at	TEXT	是	创建时间
updated_at	TEXT	是	更新时间

8.2 辅助方法

export function getAfterImage(item: FigureItem) {
  return item.images.find((image) => image.imageRole === "after")
}
export function getBeforeImage(item: FigureItem) {
  return item.images.find((image) => image.imageRole === "before")
}
export function hasBeforeAfter(item: FigureItem) {
  return Boolean(getAfterImage(item) && getBeforeImage(item))
}

13. 风险与注意事项

13.1 图片比例问题

Before 和 After 图片比例不一致时，对比效果可能不准确。

解决方向：

1. 第一版使用 object-contain 保持完整显示。
2. 后续可以增加裁剪 / 对齐工具。
3. 后续可以提示用户尽量使用同尺寸图片。

13.2 本地文件权限

Tauri 需要正确配置文件系统权限和应用数据目录读写权限。

13.3 图片路径处理

不要长期依赖用户原始图片路径。导入时必须复制到应用数据目录。

13.4 数据库迁移

虽然第一版功能简单，但数据库应预留迁移机制，避免后续多版本图片扩展时难以升级。

13.5 Canvas 缩放问题

橡皮擦对比需要处理设备像素比 DPR，否则高清屏下可能出现擦除位置偏移或画面模糊。

13.6 删除操作风险

删除收藏项会删除应用内图片副本，必须有确认弹窗。
